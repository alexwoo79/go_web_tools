# 管理员登录系统使用说明

## 🎯 功能概述

系统已添加完整的管理员登录控制，保护管理后台和数据管理 API 的访问安全。

## 🔐 默认账户

**用户名**: admin  
**密码**: admin123

⚠️ **重要提示**：首次使用后请立即修改默认密码！

## 📋 使用流程

### 1. 访问登录页面
打开浏览器访问：http://localhost:8080/login

### 2. 输入凭据
- 输入用户名和密码
- 点击"登录"按钮

### 3. 成功登录后
- 自动跳转到管理后台：http://localhost:8080/admin
- 可以看到顶部显示当前用户名和管理员标识
- 可以执行所有管理操作（查看数据、导出 CSV 等）

### 4. 退出登录
- 点击右上角的"退出登录"按钮
- 会话将被清除
- 重定向到登录页面

## 🛡️ 权限控制

### 需要管理员权限的功能

以下功能**必须登录后才能访问**：

1. **管理后台页面** (`/admin`)
   - 未登录访问会自动重定向到登录页面
   - 已登录但非管理员角色会显示"无权访问"

2. **数据导出 API** (`/api/export/{formName}`)
   - 导出 CSV 文件需要管理员权限

3. **数据查看 API** (`/api/data/{formName}`)
   - 获取 JSON 格式数据需要管理员权限

### 公开访问的功能

以下功能**无需登录即可访问**：

1. **首页** (`/`) - 表单列表页面
2. **填写表单** (`/forms/{formName}`) - 用户可以正常填写和提交表单
3. **提交数据 API** (`/api/submit/{formName}`) - 允许匿名用户提交数据

## 🔧 技术实现

### 会话管理
- 使用 Cookie + Session 机制
- Session ID 通过 SHA256 哈希生成
- 会话有效期：24 小时
- Cookie 设置 HttpOnly，防止 XSS 攻击
- SameSite=LaxMode，防止 CSRF 攻击

### 密码安全
- 密码使用 SHA256 哈希存储
- 验证时比较哈希值，不存储明文密码

### 中间件保护
使用 `RequireAdmin()` 中间件检查登录状态和角色：
```go
r.HandleFunc("/admin", h.RequireAdmin(h.AdminHandler))
r.HandleFunc("/api/export/{formName}", h.RequireAdmin(h.ExportCSVHandler))
r.HandleFunc("/api/data/{formName}", h.RequireAdmin(h.ViewDataHandler))
```

## 📊 会话管理

### 自动清理
系统每 10 分钟自动清理过期会话，释放内存。

### 并发安全
使用 `sync.RWMutex` 保证多用户并发访问时的数据安全。

## ⚙️ 扩展功能

### 添加新管理员
在 `handler.New()` 函数中添加：
```go
h.adminUsers["newadmin"] = hashPassword("password123")
```

### 修改默认密码
找到 `handler.New()` 中的初始化代码：
```go
defaultPassword := hashPassword("admin123")
h.adminUsers["admin"] = defaultPassword
```
改为你的密码的哈希值。

### 自定义会话有效期
修改 `CreateSession()` 方法中的过期时间：
```go
ExpiresAt: time.Now().Add(24 * time.Hour), // 改为其他时长
```

## 🔒 安全建议

1. **生产环境必须修改默认密码**
2. 建议使用 HTTPS 加密传输
3. 定期清理长期未使用的会话
4. 考虑添加登录失败次数限制
5. 可以考虑添加双因素认证 (2FA)

## 🐛 故障排查

### 问题 1: 无法访问登录页面
**检查**: 服务器是否正常启动  
**解决**: 查看服务器日志，确认端口监听正常

### 问题 2: 登录后仍被重定向到登录页
**可能原因**: Cookie 被浏览器阻止  
**解决**: 检查浏览器 Cookie 设置，确保允许 localhost 的 Cookie

### 问题 3: "无权访问"错误
**原因**: 当前用户不是管理员角色  
**解决**: 使用管理员账户登录，或检查用户角色设置
